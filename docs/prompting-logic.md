# Логика оптимального промта · SwapForge (v3)

Как куются эффективные промты для subject-swap — и почему именно так. Доктрина в
`server/src/engine/doctrine.ts` кодирует эти правила, сервис применяет их сам; этот документ —
чтобы понимать и проверять их глазами.

**Ревизия v3 (16.07.2026, по живым прогонам):** реконструкция сцены и длинные KEEP-инвентари
убраны — они уводили старт-кадр от оригинала и «сковывали» движение в рендере. Новый принцип ниже.

## 0. Два принципа над всеми

1. **Промт — это распределение конечного внимания модели.** Каждое слово либо усиливает нужный
   сигнал, либо разбавляет его. Плотность бьёт покрытие.
2. **Доверяй носителю мира.** Мир и движение уже ЛЕЖАТ во входе (первый кадр — для картинки,
   исходное видео — для рендера). Не пересказывай их и не перечисляй: модель следует за входом
   сама, а перечисления заставляют её «замораживать» названное — движение становится скованным
   и нереалистичным. Слова тратим только на то, чего во входе НЕТ: identity новых субъектов.

## 1. Что чем управляет (роли входов)

| Вход | Роль | Чего НЕ делает |
|---|---|---|
| первый кадр исходника | база старт-кадра: фон, ракурс, свет, композиция | НЕ источник identity |
| `video` (исходник) | мир + движение всего ролика, frame-accurate | НЕ источник identity |
| референс-фото | identity: кто человек, какая техника | НЕ источник сцены |
| промт | контракт замены | НЕ пересказ сцены |

## 2. Старт-кадр = in-place edit, не реконструкция

`images.edit` получает **первый кадр исходника ПЕРВЫМ изображением**, рефы — после. Промт
командует: «замени ТОЛЬКО человека (и технику, если она в кадре и есть реф); всё остальное —
фон, ракурс, кадрирование, свет, цвета, надписи — оставь ровно как в базовом кадре».
Реконструкция по описанию (v1–v2) уводила позу/ракурс/капшены — Seedance стартовал с неверного
кадра и тянул весь ролик в сторону. 60–120 слов, `input_fidelity=high` держит лица.

## 3. Видео-промт: минимальный контракт

Порядок фиксирован, лишнего нет:

1. **Интент, одна строка**: «Keep the entire world, background, lighting, camera work and ALL
   motion exactly as in the source video, frame for frame; completely replace [субъекты]».
   Это ВЕСЬ keep: никаких списков поверхностей, света, капшенов и таймингов.
2. **Строка ref-1, дословно**: «Reference image 1 is the exact first frame of the edit — start
   from it.»
3. **REPLACE на субъект** — здесь живёт вся детализация: «The [роль] IS the person in reference
   image 2» (жёсткая связка, не «make it look like»; роль берётся из видео — райдер, водитель,
   танцовщица, просто героиня: сервис работает с любым форматом, с техникой и без), лицо/волосы/
   возраст/одежда, identity-lock во времени («consistent and recognizable in every frame»),
   **перформанс исходника неприкосновенен** — мимика, эмоции, смех, взгляд и движение губ
   кадр-в-кадр, контакт с техникой («same posture, hands on the same grips») — если техника
   вообще свапается. Референс без пары в кадре (байк приложен, а в видео его нет) — полностью
   игнорируется в обоих промтах.
4. **LIGHT, коротко**: новых субъектов освещает свет сцены.
5. **DO NOT, одна строка**: «не меняй и не рестайль ничего, кроме [субъектов]; камеру, движение,
   тайминг и окружение не трогай».

Оверлеи при выключенной галочке «убрать текст» — одна строка «Keep all on-screen text exactly
as in the source», без цитирования капшенов. **Бюджет: 60–120 слов, потолок 150** (сервис
дожимает автокомпрессией). Опенер «Edit this video» не пишем — WaveSpeed префиксует сам.

## 4. Подавители рисков — только в итерациях

Карта рисков из анализа больше НЕ вливается в первый промт (это тоже «сковывание»). Она
включается точечно при итерации, когда пользователь отметил конкретный артефакт: артефакт →
его рычаг (`shared/taxonomy.ts`):

| Артефакт | Симптом | Фикс |
|---|---|---|
| identity_bleed | лицо-гибрид | усилить IS/completely replace + больше ракурсов рефа |
| world_drift | сцена поехала | точечный KEEP именно уехавшего + DO NOT |
| temporal_drift | морф по ходу клипа | re-assert «in every frame»; длинный клип — резать |
| pasted_on | «наклейка» | усилить LIGHT конкретикой света сцены |
| cross_wiring | реф не туда | пересобрать нумерацию/порядок массива |

Чинится только сломанное. 👍-генерации становятся few-shot якорями уровня (не длины: бюджет
главнее примера).

## 5. Нумерация референсов

ref 1 = старт-кадр, дальше человек, потом техника — ровно в порядке массива `reference_images`.
Перепутанный порядок = cross-wiring (лицо на баке). Заметка пользователя к рефу главнее фото.

## 6. Форматы — не упоминаются нигде

Ни в промтах, ни в notes, ни в UI: никаких «9:16», «720p», «2K». Выход сервиса всегда один
(720p, вертикаль) — это константа продукта, а не информация для промта или пользователя.

## 7. Половина результата — референсы

Лицо фронтально при хорошем свете + фулл-боди с нужным аутфитом; техника — чистый 3/4;
2–4 ракурса (или реф-лист «все ракурсы») лочат identity сильнее любых слов.

## 8. Чек-лист готового промта (20 секунд глазами)

- [ ] videoPrompt: интент-строка без перечислений; ref-1 дословно; REPLACE с «IS the person…»,
      lock «in every frame», контакт с техникой; LIGHT; одно DO NOT-предложение
- [ ] Никаких списков объектов сцены, капшенов, таймингов, разрешений и форматов
- [ ] 60–120 слов (imagePrompt тоже)
- [ ] imagePrompt командует edit базового кадра: «replace only…, keep everything else exactly»
- [ ] Нумерация промта = порядок reference_images
