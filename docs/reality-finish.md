# Reality Finish — адаптивный camera/UGC-финиш готового рендера

Финальный этап после рендера: делает ролик менее «ИИ-гладким» — как съёмка на телефон
или реальную камеру. Локальный ffmpeg, без GPU и внешних API; для пользователя бесплатно
(закрыт дневными капами `LIMIT_FINISH_PER_DAY` / `LIMIT_FINISH_PREVIEW_PER_DAY`).

## Режимы

| Режим | Характер | Ключевые фильтры |
|---|---|---|
| `natural` | минимальная обработка: лёгкое зерно, естественная резкость, мягкая цветокоррекция | `eq` → `unsharp` → `noise` |
| `phone` | видео со смартфона: сенсорный шум (люма+хрома), телефонная резкость, живой контраст **+ реальная компрессия** (crf 20–24 растёт с интенсивностью) | `eq` → (`curves`) → `unsharp` → `noise` |
| `camera` | камера блогера: мягкие света (rolloff), приподнятые тени, плотный тёплый цвет, киношное зерно, меньше цифровой резкости | `eq` → `curves` → `colorbalance` → (`gblur`) → `noise` |

## Адаптация под ролик (не статичные пресеты)

Перед применением рендер замеряется одним проходом ffmpeg (6 веток `filter_complex`,
~16 кадров равномерно по всему ролику, `server/src/engine/finish.ts`):

- яркость/контраст/насыщенность — `signalstats` (YAVG, YLOW/YHIGH, SATAVG);
- пережжённые света / проваленные тени — `lutyuv`-маски порогов 234/18;
- резкость — энергия границ `sobel`;
- шум/текстура — residual после `gblur` (`blend=difference`);
- кожа — `geq`-маска YCbCr (Cb 77–127, Cr 133–173).

Правила `computeFinishPlan` (чистая функция, покрыта тестами):

- зерно гасится пропорционально уже имеющемуся шуму (шумному ролику — минимум);
- насыщенность тянется к **целевой** для режима: перенасыщенный ролик приглушается, а не усиливается;
- при проваленных тенях яркость не опускается никогда — только лёгкий подъём;
- резкость не добавляется уже резкому видео; blur camera-режима пропускается для мягкого;
- при доле кожи >12% — кап на прибавку насыщенности, тепло `colorbalance` ополовинивается;
- интенсивность (0.1–1.0, шаг 0.1) линейно масштабирует все дельты.

## Контракт «меняется только внешний вид»

Обработка 1:1 по кадрам: **никакого** изменения движения, длительности, fps
(`-fps_mode passthrough`), разрешения, AR, кадрирования; **аудио — потоковая копия**
(`-c:a copy`, проверено md5 бит-в-бит); никакой искусственной тряски. После применения
результат валидируется probe-ом (размеры/длительность/fps/наличие аудио) — при
расхождении файл не публикуется, статус `failed`.

## Поток

1. `POST /api/generations/:genId/finish/preview {mode, intensity}` — замер (кэш в
   `generations.finish_json.analysis`) + два коротких превью (~2.4с из середины,
   `preset veryfast`): `before`/`after` в `projectDir/finish/`, кэш по бакету
   интенсивности; отдаются через media-sub `finish`. Кэш-хиты не тратят дневной кап.
2. `POST /api/generations/:genId/finish` — асинхронное применение ко всему ролику
   (серийная очередь, `preset fast`); статус едет в `GenerationRow.finish`
   (processing/done/failed), фронт поллит. Результат — `renders/gen_<id>_finish.mp4`
   (`generations.finish_file`), **оригинал остаётся** и скачивается как раньше.
3. `DELETE /api/generations/:genId/finish` — убрать обработку (замер остаётся).

Restart-safe: повисшие `processing` на буте помечаются `failed` (`resumeFinishJobs`).
Уборка: кэш превью живёт 48ч / до удаления генерации (`sweepTransientProjectFiles`);
ротация рендера уносит и finish-артефакты (`enforceLatestResultLimit`/`enforceStorageCap`).

UI: экран «Готовый ролик» → блок «📷 Reality Finish» — 3 режима, Before/After на
фрагменте, слайдер интенсивности, «Применить ко всему ролику», «Убрать обработку».
