# Release evidence

Для каждого кандидата создаётся локальный/операторский каталог `release-evidence/<exact-sha>/`.
Git игнорирует содержимое этих каталогов: подтверждение нельзя переносить между коммитами.

Обязательные файлы перечислены в `deploy/launch-gate.sh`. Первая строка каждого файла должна
быть exact SHA, следующие строки — дата, оператор, среда и ссылка/путь к доказательству.

Не создавайте evidence «авансом»: файл означает, что соответствующая проверка реально выполнена.
